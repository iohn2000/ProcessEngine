To configure a workflow you have to edit the XML Workflow Definition.

Process:

1. Checkout:
To edit a workflow, you have to checkout a workflow. In the checked-out state you can change the XML and save it, to don't loose any changes.
No other user is allowed to checkout a already checked-out workflow.
2. Save:
The edited XML must be saved before doing any other action, otherwise your changes are lost.
3. Checkin:
Only checked-in workflows are taken for new workflow processes. If you check-in a workflow the version number will be increased.
All versions are saved in the database.
Undo Checkout:
If you don't want to commit any changes to a new workflow process, you can undo your changes with a undo checkout
